# MCode 编辑器整体框架分析

MCode 编辑器是基于 VS Code (Monaco Editor) 的二次开发，用于深度集成人工智能（AI）辅助编程功能。它主要集成了 AI 聊天（Chat/Sidebar）、代码流式生成/快捷编辑（Ctrl+K）、代码自动补全（Autocomplete）和模型上下文协议（MCP）工具。

## 1. 架构模式

MCode 遵循 Electron 及其底层的 VS Code 设计哲学，采用**多进程架构**，并将核心代码划分在 `src/vs/workbench/contrib/mcode/` 目录下。

```mermaid
graph TD
    subgraph Browser Process (渲染进程)
        ReactUI[React 界面: Sidebar, Settings] -->|调用| BrowserServices[前端服务: editCodeService, autocompleteService]
        BrowserServices -->|IPC 通信| SendLLMMessageService[sendLLMMessageService]
    end

    subgraph Electron Main Process (主进程)
        SendLLMMessageService -->|IPC 管道| sendLLMMessageChannel[sendLLMMessageChannel]
        sendLLMMessageChannel -->|具体实现| sendLLMMessageImpl[sendLLMMessage.impl.ts]
        sendLLMMessageImpl -->|网络请求| Providers[LLM 提供商 API: OpenAI, Anthropic, Ollama, etc.]
    end
```

### 1.1 渲染进程（Browser）
* **职责**：处理所有与 UI 相关的渲染（React 组件）、用户按键监听（Actions/Commands）、Monaco Editor 装饰器（Decoration Zone, Diff 区域）以及行内补全的显示。
* **限制**：出于安全考虑（CSP），渲染进程无法直接访问本地 Node.js 模块，且由于安全域限制，直接请求本地部署的 LLM（如 Ollama）或第三方 API 会遭遇跨域（CORS）问题。

### 1.2 主进程（Electron-Main）
* **职责**：处理网络通信（调用外部 LLM API）、文件系统读写、MCP 进程的拉起和通信等 Node.js 原生底层逻辑。
* **优势**：具有完整的 Node.js 环境，且可以随意引入第三方 SDK（如 `@anthropic-ai/sdk`, `openai` 等），不受浏览器沙箱与 CSP 的限制。

### 1.3 进程间通信（IPC）
* MCode 通过继承 VS Code 的 `IChannel` / `IServerChannel` 机制，实现了极简且强类型的 IPC 管道。
* 前端通过 `sendLLMMessageService.ts` 调用，后端通过 `sendLLMMessageChannel.ts` 接收并最终执行 `sendLLMMessage.impl.ts`。

## 2. 核心三层目录划分

在 [src/vs/workbench/contrib/mcode/](file:///d:/work/void/src/vs/workbench/contrib/mcode) 目录下，代码清晰地划分为了以下三层：

1. **`browser/`**：
   - 包含前端 React UI 代码（[react/](file:///d:/work/void/src/vs/workbench/contrib/mcode/browser/react)）和 VS Code Workbench 的各种交互服务（如编辑服务、自动补全服务）。
2. **`common/`**：
   - 存放跨进程的数据结构、接口定义、共享的工具类（如配置服务类型、模型基准能力数据 [modelCapabilities.ts](file:///d:/work/void/src/vs/workbench/contrib/mcode/common/modelCapabilities.ts)）。
3. **`electron-main/`**：
   - 存放与系统或 Node.js 环境交互的主进程通道实现（如 LLM 发送实现、MCP 长连接管理）。

## 3. 全局入口与注册

项目的注册和依赖注入统一在 [mcode.contribution.ts](file:///d:/work/void/src/vs/workbench/contrib/mcode/browser/mcode.contribution.ts) 中管理。
VS Code 使用 `@createDecorator` 定义依赖项，并使用 `registerSingleton` 将服务注册为单例模式（Singleton Service），从而可以在任何构造函数中直接通过属性装饰器进行注入：
* 例如，[IVoidSettingsService](file:///d:/work/void/src/vs/workbench/contrib/mcode/common/mcodeSettingsService.ts) 是一个全局单例，任何 UI 和后台处理组件都可以读取或修改当前的用户配置。
