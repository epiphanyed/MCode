# 本地向量模型（Ollama）支持 - RAG 索引与检索方案设计

本方案旨在为 MCode 设计并实现对本地向量嵌入模型（通过 Ollama）的支持，以替代目前写死的 OpenAI 线上向量模型。这将实现完全离线的 RAG 代码库索引与检索问答，并在设置界面中提供独立的配置模块。

## 核心设计决策

> [!IMPORTANT]
> **Ollama 本地环境要求**
> 使用本地向量化时，用户必须在本地（或局域网内）运行 Ollama 服务，并提前拉取好向量模型（推荐使用 `ollama pull nomic-embed-text`，这是目前最轻量、效果最好的专用向量模型之一）。
>
> **动态维度自适应（Dynamic Dimension Detection）**
> 不同的向量模型输出的向量维度完全不同（例如 OpenAI 为 1536 维，Ollama 的 `nomic-embed-text` 为 768 维，`mxbai-embed-large` 为 1024 维）。
> 为了避免维度冲突（尤其是在 Milvus 中），系统将在初始化索引时**自动发送一条单字测试文本进行试算**，动态获取当前模型的向量维度，并自动适配本地及 Milvus 的数据表维度配置。

## 详细变更方案

### 1. 全局配置与数据结构设计

#### [修改] [mcodeSettingsTypes.ts](file:///d:/work/void/src/vs/workbench/contrib/mcode/common/mcodeSettingsTypes.ts)
- 在 `GlobalSettings` 接口和 `defaultGlobalSettings` 默认配置中引入以下三个新字段：
  - `embeddingProvider`: `'openai' | 'ollama'`（嵌入模型提供商，默认 `'openai'`）
  - `embeddingModel`: `string`（模型名称，默认 `'text-embedding-3-small'`；若选择 Ollama，默认设为 `'nomic-embed-text'`）
  - `ollamaEndpoint`: `string`（Ollama 的服务地址，默认 `'http://127.0.0.1:11434'`）

#### [修改] [mcodeSettingsService.ts](file:///d:/work/void/src/vs/workbench/contrib/mcode/common/mcodeSettingsService.ts)
- 在读取和初始化持久化配置时（`readAndInitializeState`），如果发现上述三个字段未定义，则自动用默认值填充，保证版本平滑升级。

### 2. RAG 服务与本地模型适配

#### [修改] [llamaIndexService.ts](file:///d:/work/void/src/vs/workbench/contrib/mcode/electron-main/rag/llamaIndexService.ts)
- **自定义 Ollama 向量模型类**：
  利用项目中已有的 `ollama` 官方 npm 依赖包，实现一个继承自 LlamaIndex `BaseEmbedding` 的自定义向量类 `CustomOllamaEmbedding`。它通过 `ollama.embed` 接口与本地 Ollama 服务交互以生成向量。
- **动态实例化 `Settings.embedModel`**：
  在 `initializeIndex` 中，读取前端传入的 `embeddingProvider` 和 `embeddingModel`：
  - 如果为 `'openai'`：实例化原有的 `OpenAIEmbedding`（依赖环境变量中的 `OPENAI_API_KEY`）。
  - 如果为 `'ollama'`：实例化我们自定义的 `CustomOllamaEmbedding`，指向配置的 `ollamaEndpoint`。
- **自动检测维度**：
  在主进程初始化阶段，调用 `await Settings.embedModel.getTextEmbedding("test")` 获取向量，通过 `length` 动态计算出模型维度 `dimensions`。
- **动态应用维度**：
  将计算出的维度动态传入 Milvus 集合创建参数（`dim: dimensions`）和本地索引配置中，彻底杜绝维度不匹配导致的系统崩溃。

#### [修改] [mcodeRagMainService.ts](file:///d:/work/void/src/vs/workbench/contrib/mcode/electron-main/mcodeRagMainService.ts) & [mcodeRagService.ts](file:///d:/work/void/src/vs/workbench/contrib/mcode/browser/mcodeRagService.ts) & [mcodeRagTypes.ts](file:///d:/work/void/src/vs/workbench/contrib/mcode/common/mcodeRagTypes.ts)
- 修改 `initializeIndex` 接口的方法签名，使其能够接收从前端 UI 传入的嵌入模型配置参数（`embeddingProvider`、`embeddingModel`、`ollamaEndpoint`）。

#### [修改] [chatThreadService.ts](file:///d:/work/void/src/vs/workbench/contrib/mcode/browser/chatThreadService.ts)
- 在系统自动初始化 RAG 时，从全局配置服务读取对应的嵌入模型参数，并传入 `this._mcodeRagService.initializeIndex`。

### 3. 设置界面（UI）改造

#### [修改] [Settings.tsx](file:///d:/work/void/src/vs/workbench/contrib/mcode/browser/react/src/mcode-settings-tsx/Settings.tsx)
- 在原有的“Index Settings”页面中，紧跟在“Index Type”之后增加一个全新的 **“Embedding Model Configuration（嵌入模型配置）”** 模块。
- **UI 组件设计**：
  - **Embedding Provider**：提供 Segment Control（分段按钮），供用户在 `Cloud (OpenAI)` 和 `Local (Ollama)` 之间切换。
  - **Embedding Model**：输入框，提供输入提示（OpenAI 提示使用 `text-embedding-3-small`，Ollama 提示使用 `nomic-embed-text`）。
  - **Ollama Endpoint**：输入框，在选择 `Local (Ollama)` 时可见，允许修改本地端点（如 `http://127.0.0.1:11434`）。
  - 提供 **“Save & Apply Embedding Settings”** 按钮以保存并实时重新初始化 RAG。

## 验证与测试计划

### 手动功能验证步骤
1. **Ollama 本地索引测试**：
   - 本地启动 Ollama 并运行 `ollama pull nomic-embed-text`。
   - 打开 MCode 设置，将嵌入提供商切换为 **Ollama**，模型设为 `nomic-embed-text`。
   - 点击 **Save & Apply Settings**。
   - 验证界面状态栏是否成功转为绿色并显示成功。
    - 验证本地 `%APPDATA%/MCode/LlamaStore/` 下新生成的 SQLite 数据库及 HNSW 索引（`<WorkspaceStoreName>.db` 与 `<WorkspaceStoreName>.usearch`）中的向量数据维度是否为 768 维。
   - 在聊天框中提问，验证 RAG 能够精准拉取本地代码上下文并回复。
2. **OpenAI 兼容性回归测试**：
   - 配置 `OPENAI_API_KEY` 环境变量。
   - 设置中将提供商切换回 **OpenAI**，模型设为 `text-embedding-3-small`。
   - 验证索引正常重建且维度自动适配为 1536 维。
3. **Milvus + Ollama 混合测试**：
   - 启动本地 Milvus 容器。
   - 启用 Milvus，并选择 Ollama 的 `nomic-embed-text` 作为向量源。
   - 验证 Milvus 中自动成功创建了对应维度的 Collection，并能进行向量写入与混合检索。
